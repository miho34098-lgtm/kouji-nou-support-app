## 2. セキュリティの制約

| ID | 制約 | 根拠 |
|---|---|---|
| C-SEC-001 | APIキー・Firebase設定・VAPID鍵のコード直書き禁止。環境変数（.env）経由のみ | SDD CC-03 |
| C-SEC-002 | XSS対策必須。全ユーザー入力（プロフィール自由記述等）は表示時にエスケープ | OWASP Top 10 |
| C-SEC-003 | 入力検証必須。クライアント側とFirestore Security Rulesの両方で検証 | SDD CC-04 |
| C-SEC-004 | 依存パッケージの既知脆弱性チェック必須（`npm audit`） | サプライチェーン攻撃防止 |
| C-SEC-005 | 本人/介護者のロールに応じたFirestore Security Rulesでのアクセス制御必須。介護者は紐付けられた本人データのみアクセス可 | SRS §11 |

### プロジェクト固有
| C-SEC-101 | **【Round2改訂】障害プロフィール等の要配慮個人情報を収集する機能の追加を禁止する**（Round1では同意フロー必須としていたが、Round2でパーソナライズ機能自体を廃止したため「そもそも収集しない」制約に変更） | 個人情報保護法・SaMD該当性回避、開発時間最小化（社長要望） |
| C-SEC-102 | 訓練メニュー関連のいかなる機能も疾病名・重症度スコアを算出・表示してはならない。個人の障害種類に基づく絞り込み・パーソナライズ機能の追加も禁止する | FR-TRN-005、SaMD-3制約、Round2見直し |

## 3. データ管理の制約

| ID | 制約 | 根拠 |
|---|---|---|
| C-DATA-001 | 本番Firestoreへの破壊的操作（コレクション全削除等）禁止。Emulatorでの検証を経てから実施 | H1 DRY_RUN |
| C-DATA-002 | 個人情報（PII、要配慮個人情報）のログ出力禁止 | プライバシー保護 |
| C-DATA-003 | テストデータに実在の障害プロフィール情報を使用禁止。ダミーデータのみ使用 | プライバシー保護 |

### プロジェクト固有
| C-DATA-101 | **Firebase Blazeプランへのアップグレード禁止**。請求先アカウントを紐付けない。**【Round1修正 OPUS-006】月次で①Firebaseコンソールの請求先アカウント未設定確認②Cloudflareダッシュボードでの無料枠利用状況確認を実施し、PROGRESS.mdに記録する（担当者はPROGRESS.md「運用責任者」欄参照）** | ADR-005、運用コストゼロ制約 |
| C-DATA-104 | Firebaseサービスアカウント鍵はGitHub Actions Secretsで管理し、年1回を目安にローテーションする【Round3改訂: wrangler secret→GitHub Actions Secrets】 | ADR-010、CX-18 |
| C-DATA-105 | GitHub Actionsのscheduleワークフローが60日間無活動で自動無効化されないよう、月次でリポジトリへのpushまたはworkflow_dispatch手動実行を行う | HARNESS.md H6（Round3新規） |
| C-DATA-102 | ProxyLogは90日でTTL削除する設定を必ず維持する | FR-ADM-002、Firestore無料枠維持 |
| C-DATA-103 | 訓練メニューのコンテンツ変更は`src/data/trainingMenus.json`の静的編集のみで行い、動的生成ロジックを追加しない | ADR-006 |

## 5. AI（Claude Code）への制約

| ID | 制約 | 根拠 |
|---|---|---|
| C-AI-001 | SRS未記載機能の追加禁止 | スコープクリープ防止 |
| C-AI-002 | テスト削除・改ざん禁止 | 品質保証 |
| C-AI-003 | 50往復超のセッション継続禁止 | docs/harness/session-mgmt.md |
| C-AI-004 | 「完了」宣言はDoD全項目チェック後のみ | CLAUDE.md PART B |
| C-AI-005 | force push禁止 | 変更追跡性 |
| C-AI-006 | CONSTRAINTS.md自体の変更禁止（人間承認後のみ） | 自己改変防止 |

### プロジェクト固有
| C-AI-101 | 訓練提案機能に「治療」「診断」「改善を保証」等の医療行為を想起させる文言を追加禁止 | SaMD-4制約 |
| C-AI-103 | 訓練メニュー画面の免責表示から「無理をせず休んでよい旨」「専門家（主治医・作業療法士等）へ相談するよう促す旨」を削除・弱体化することを禁止 | FR-TRN-004、社長要望(2026-07-14) |
| C-AI-102 | Cloudflare Workers/Firestoreの無料枠を超過する設計変更（高頻度ポーリング等）を提案する場合は必ず事前に人間へ確認 | 運用コストゼロ制約 |

# 採用停止基準表

| 技術/サービス | 採用停止条件 | 代替案 |
|---|---|---|
| GitHub Actions（NotificationCheckJob）【Round3改訂】 | プライベートリポジトリ化により月2,000分の無料枠を継続的に超過する。またはscheduleワークフローが60日無活動で無効化された状態を1週間以上放置した場合 | パブリックリポジトリ維持（無料枠実質無制限）、または有料プランへの移行。無効化対策はHARNESS.md H6参照 |
| Firebase Firestore（Sparkプラン） | 読取5万/書込2万の無料枠を継続的に超過する。または誤ってBlazeプランに切り替わった場合 | Sparkプラン相当の他無料DB（Supabase等）への移行を検討。Blaze切替時は即座に予算アラート設定 |
| GitHub Pages【Round3改訂】 | 月100GBのソフト帯域上限を継続的に超過する、または商用利用とみなされる警告を受けた場合 | Cloudflare Pages無料枠（帯域実質無制限）への移行を再検討 |
| web-push (Node.js相当ロジック) | 重大なセキュリティ脆弱性が修正されないまま3ヶ月以上放置された場合 | 自前VAPID実装への切替（RFC 8292に基づく） |
| Dexie.js | メンテナンスが12ヶ月以上停止した場合 | 素のIndexedDB APIまたは後継OSSへの移行 |
| 訓練メニューの参照元ガイドライン | 国立障害者リハビリテーションセンターの公開資料が改訂・非公開化された場合 | 改訂版の内容を精査し訓練メニューを更新。非公開化時は代替の公的情報源を再調査 |
